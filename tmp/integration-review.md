## domain:pipeline

### Finding 1 — New fallback pattern is unanchored (pre-existing style)

**Severity:** LOW
**Path:** `packages/cisco_asa/data_stream/log/elasticsearch/ingest_pipeline/default.yml`
**Line:** 471

**Problem:**
The new fallback grok pattern (line 471), like the pre-existing primary pattern (line 470), lacks `^...$` anchors. Without anchors the regex engine scans the entire input string for a partial match, which is slower and can produce unexpected captures on unusual log variants.

```yaml
# current (unanchored — both primary and fallback)
patterns:
- "AAA user authentication Rejected(%{SPACE})?: reason = ..."
- "AAA user authentication Rejected(%{SPACE})?: reason = ... %{DATA:source.user.name} ..."
```

This is a pre-existing pattern in the file — the new line is consistent with existing convention. Anchoring would require updating all `parse_113015` patterns together (out of scope for this fix).

**Recommendation:**
Track as a follow-up improvement. When revising this processor in future, add `^` and `$` to both patterns simultaneously.
